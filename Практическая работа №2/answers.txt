1. Чем семантическая верстка отличается от несемантической и почему она важна?
Семантическая верстка использует теги по смыслу (header, article, nav), а несемантическая - только div. Важна для доступности и SEO моего цветочного магазина.

2. В каких случаях следует использовать <article> вместо <section>? Приведите примеры из вашей работы.
<article> - для независимого контента. В моей работе: статья "Сезонные предложения". <section> - для группировки, например, "Наши услуги".

3. Для чего предназначены теги <header> и <footer> и могут ли они использоваться несколько раз на странице?
<header> - для шапки, <footer> - для подвала. Да, используются многократно. В работе: главные header/footer и дополнительные внутри article.

4. Какой тег следует использовать для основного контента страницы и почему не стоит использовать для этого <div>?
<main>. В него помещены статьи и услуги магазина. <div> не семантичен, не передает смысл для скринридеров.

5. В чем разница между <nav> и обычным списком ссылок? Когда обязательно использовать <nav>?
<nav> - для основной навигации. В работе: главное меню. Обычный список - для популярных товаров в aside. Обязательно для главного меню.

6. Для чего используется атрибут datetime в теге <time> и почему он важен?
Для машиночитаемого формата даты. В работе: дата публикации статьи "15 октября 2023". Важен для поисковиков.

7. Какие преимущества дает использование <figure> и <figcaption> вместо простого <img> с подписью?
Семантически связывают изображение с подписью. В работе: фото осеннего букета с описанием и ценой. Улучшает доступность.

8. Когда следует использовать <aside> и какой контент в него помещать?
Для косвенно связанного контента. В работе: популярные товары, время работы, форма подписки, таблица поставок цветов.

9. Как семантическая разметка влияет на SEO и доступность веб-страницы?
Улучшает SEO - поисковики видят структуру магазина. Повышает доступность - скринридеры правильно озвучивают навигацию и контент.

10. Какие теги вы использовали для улучшения структуры документа и как они помогают пользователям и поисковым системам?
header, nav, main, article, section, aside, footer, figure, figcaption, time. Помогают понять структуру страницы цветочного магазина пользователям и поисковым системам.
