1. В чем разница между селектором по классу и по ID с точки зрения специфичности и возможности повторного использования?
Селектор по классу имеет специфичность (0,1,0), по ID - (1,0,0). Классы можно использовать многократно, ID должен быть уникальным на странице.

2. Как работает каскад в CSS и в каком порядке применяются стили при конфликте правил?
Браузер применяет стили в порядке: таблицы стилей, специфичности, порядка объявления. При конфликте побеждает правило с большей специфичностью, при равенстве - последнее объявленное.

3. Рассчитайте специфичность для селекторов:
	ul li.important
	#site-title
	.content > .featured-post
ul li.important - (0,1,2)
#site-title - (1,0,0)
.content > .featured-post - (0,2,0)

4. Какие свойства в CSS наследуются по умолчанию, а какие нет? Приведите по 3 примера каждого типа.
Наследуются: color, font-family, line-height. Не наследуются: margin, padding, border.

5. Как работает селектор > (дочерний селектор) и чем он отличается от селектора потомков (через пробел)?
Селектор > выбирает только прямых потомков, а через пробел - всех потомков любого уровня вложенности.

6. Для чего используются псевдоклассы :hover и :nth-child()? В каких практических задачах они могут пригодиться?
:hover для стилей при наведении, :nth-child() для выбора элементов по порядку. Применяются для интерактивности и стилизации таблиц/списков.

7. Что делает правило !important и почему его следует использовать с осторожностью?
Принудительно применяет стиль, игнорируя специфичность. Опасно тем, что усложняет переопределение стилей и нарушает каскад.

8. Как псевдоэлементы ::before и ::after расширяют возможности CSS и какой контент они могут содержать?
Добавляют декоративный контент до/после элемента. Могут содержать текст, иконки, оформление через content, но не HTML.

9. В каком порядке браузер применяет CSS-правила при одинаковой специфичности селекторов?
Побеждает правило, объявленное последним в CSS-коде.

10. Как инструменты разработчика в браузере помогают анализировать и отлаживать CSS-код?
Показывают примененные стили, специфичность, наследование, позволяют редактировать CSS в реальном времени для тестирования.
